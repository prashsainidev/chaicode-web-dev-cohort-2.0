## Timeline
* **Start:** 28 Feb 2026, 8:00 pm

## Description
In real engineering roles, you are expected to:
* Understand a product problem
* Design database schemas
* Build complete workflows
* Handle roles and permissions
* Deploy production-ready applications

This hackathon is designed to test exactly that.

You will be given abstract product problems. You are expected to research similar platforms in the market and design your own solution. Do not copy an existing product. Build your own version with proper logic and structure.

## Rules and Technical Requirements
* This hackathon is optional.
* The project must be built individually. No teams allowed.
* Prefer to keep your code in MERN stack only
* You can use external SDKs such as firebase, supabase, convex, etc

## Choose One Problem Statement
You must select only one of the following projects. Do not combine problem statements. Do not work in groups.
The problems are intentionally open-ended but feature-rich so they can be demonstrated properly.

### Problem Statement 1: Local Services Booking Platform
Build a platform that connects customers with local service professionals and manages the complete job lifecycle.

#### Core Requirements
**Customer Side**
* Browse services by category
* Filter by city or area (text-based)
* Create booking request (address, date/time, notes, optional image upload)
* Display pricing before confirmation
* Track booking status: `Requested → Confirmed → In-progress → Completed → Cancelled`
* Reschedule or cancel booking
* Submit rating and review

**Service Provider Side**
* Create and manage professional profile
* Toggle availability status
* Accept or reject bookings
* Update job status
* Add work notes and upload before/after images

**Admin Side**
* Approve service providers
* Manage service categories
* Moderate reviews

*Focus on workflow logic and state transitions.*

### Problem Statement 2: Rental Listings & Move-in Platform
Build a rental housing platform that manages property discovery, visit scheduling and move-in workflows.

#### Core Requirements
**Tenant Side**
* Browse listings with filters (location text, budget range, move-in date)
* View detailed property page (gallery, amenities, rules, availability timeline)
* Request property visit
* Track visit status: `Requested → Scheduled → Visited → Decision`
* Shortlist properties
* Compare 2–3 properties side-by-side

**Operations Features**
* Move-in checklist (document uploads, agreement confirmation, inventory list)
* Support ticket system with threaded messages
* Request stay extension

**Admin Side**
* Review and publish listings
* Manage support tickets
* Change listing status: `Draft → Review → Published`

*Focus on dashboards, structured workflows and proper schema design.*

### Problem Statement 3: Collaborative Trip Planning
Build a collaborative itinerary planning platform with shared access and organization tools.

#### Core Requirements
**Trip Planning**
* Create trip (title, dates, travelers)
* Day-wise itinerary builder
* Add activities as cards
* Reorder activities

**Collaboration**
* Invite members
* Role-based access (Owner, Editor, Viewer)
* Comment system per day or activity

**Organization**
* Checklists (packing, to-do, etc.)
* File attachments (tickets, PDFs, images)
* Manual reservation entries
* Budget tracking with expense summary

*Focus on collaboration logic, permissions, and clean data modeling.*
