## Instructions
In this assignment, you will explore the Linux file system beyond basic commands and write a short blog based on your discoveries. This is not a command-practice task. This is a Linux hunting exercise.

Your goal is to investigate how Linux actually works under the hood through its file system structure and related components like networking configs, DNS settings, routing tables, logs, processes, permissions, and system behavior.

Focus on interesting findings, not on listing commands.

**Examples of directions you can explore:**
- How `/etc` controls system behavior
- Where DNS configuration lives and how it works
- Routing table inspection through system files
- Network interface configuration locations
- System logs and what insights they provide
- User management files
- Permission structures and security implications
- Process-related filesystem entries inside `/proc`
- Device handling inside `/dev`
- Boot-related configs inside `/boot`
- Service configs inside `/systemd` or `/etc/systemd`
- Environment configuration behavior

Think like a system investigator. Your blog should explain what you discovered and why it matters.

## Submission Instructions
Write a short technical blog on Hashnode or similar platform covering:

- At least 5-10 meaningful findings from Linux file system exploration
- Each finding must explain:
  - What the file/folder does
  - Why it exists
  - What problem it solves
  - What interesting insight you learned from it
- Include commands only if required for explanation
- Avoid basic command tutorials like: `ls`, `cd`, `mkdir`, `pwd`

*This is not a beginner command assignment. This is a discovery assignment.*
