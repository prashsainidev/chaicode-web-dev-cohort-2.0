# Backend Engineering - Chai Aur SQL (Part 2)

## Overview

This backend class was held on Sunday, `05 Apr 2026`.

The topic for this session was `Chai Aur SQL (Part 2)`.

This class continued the SQL journey from the previous day and moved into relational queries, transactions, schema design, and a booking-system mini project.

---

## What I Learned

### Joins

- using `INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, and `FULL OUTER JOIN`
- understanding how related tables are connected through query logic
- reading data across multiple tables instead of treating tables in isolation

---

### Performance And Data Integrity

- why indexes help query performance
- why transactions matter when multiple steps must succeed together
- how ACID ideas like atomicity, consistency, isolation, and durability protect data

---

### Schema Thinking

- designing a larger social-media-style schema
- thinking beyond single-table exercises
- connecting database design with application behavior

---

### Mini Project

- building a booking-system mini project with `express` and `pg`
- using connection pooling in PostgreSQL
- applying transactions with seat booking logic
- understanding why row locking and safe updates matter in real apps

---

### Post-Class Practice

- continuing SQL schema practice through a `5-day DB design series`
- keeping the day-wise ERD assignments inside `assignments/` under this class
- using the series to practice business-first relational modeling beyond the live class examples
