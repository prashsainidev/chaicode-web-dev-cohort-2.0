# Database Design Series - April 2026

This directory contains my 5-day database design series work from the Web Dev Cohort 2026. I organized each assignment into a separate day-wise folder so the progression stays clear and the repo remains easy to review.

Across this series, my focus was not just to draw ER diagrams, but to understand each business problem first and then model it through entities, relationships, PKs, and FKs. I tried to keep the designs practical, readable, and realistic so that the business flow jaldi samajh aa sake.

## Day-wise Work

1. `day-01-instagram-thrift-creator-store/` covers a thrift and handmade creator store.
2. `day-02-fitness-influencer-coaching-platform/` covers an online coaching platform.
3. `day-03-clinic-appointment-and-diagnostics-platform/` covers appointment, consultation, diagnostics, and billing flow.
4. `day-04-comic-con-parking-system/` contains the original parking assignment and a separate improved version after peer review.
5. `day-05-smart-elevator-control/` covers a multi-building elevator control system.

## What Each Day Folder Includes

Each assignment folder keeps the editable ERD source in `eraser-diagram.txt`. The exported board image is stored as `diagram.png` once the final ER diagram is exported. A short `README.md` is included to explain the design approach and the main modeling decisions.

## Approach

I intentionally kept these ERDs simple enough to read quickly, but detailed enough to show proper business understanding. The idea was to avoid unnecessary complexity and still make the schema feel usable in a real system, taaki peer review me clarity bani rahe.
