What is the DOM?

🏗️ Real-life analogy: Socho tumhara HTML ek blueprint hai aur DOM wo actual building hai jo browser us blueprint se banaata hai. Blueprint change karo toh building nahi badlegi — lekin DOM ko JavaScript se manipulate karo toh page turant update ho jaata hai. DOM = Browser ki living, breathing memory of your page.
📄

HTML vs DOM

HTML is raw text. DOM is the object that browser creates from that text in memory. They start same but diverge.

🌳

Tree Structure

DOM is a tree of nodes. Every tag is a node. Text inside tags is also a node. Everything has a parent-child relationship.

🔗

Live Object

DOM is live. Change it with JS and the browser re-renders instantly — no page reload needed.

🌐

Browser API

DOM is a browser API — not part of JavaScript itself! JS uses it, but DOM lives in the browser (C++ under the hood via WebIDL).

document
└─<html>
├─<head>
└─<title> "My Page" </title>
└─<body>
├─<h1 id="title"> "Hello" </h1>
└─<div class="box">
├─<p> "Paragraph" </p>
└─<button> "Click Me" </button>
💡 Memory Trick
DOM ko yaad rakho: Document Object Model = Browser ka ek JavaScript object hai jisme poora page stored hai as a tree. typeof document === 'object' ✅

Types of Nodes

🧱 Analogy: DOM tree mein sirf HTML tags nahi hote — bohot types ke "bricks" hote hain. Element nodes toh buildings hain, lekin unke beech text bhi hai, comments bhi, aur document khud ek special root node hai. Har node ka ek nodeType number hota hai — browser isi se identify karta hai.
nodeType Name Example Kitna Common?
1 Element Node <div>, <p>, <h1> ⭐⭐⭐ Most common
2 Attribute Node class="box", id="btn" Deprecated — use getAttribute()
3 Text Node Text inside any tag, whitespace bhi! ⭐⭐⭐ Very common
4 CDATASection Node XML mein hota hai, HTML mein nahi Rare (XML only)
7 ProcessingInstruction <?xml version="1.0"?> Rare
8 Comment Node <!-- ye comment hai --> ⭐⭐ Moderate
9 Document Node document object itself ⭐⭐⭐ Root of everything
10 DocumentType Node <!DOCTYPE html> ⭐ Seen in interviews
11 DocumentFragment Node document.createDocumentFragment() ⭐⭐ Performance use
💡 Sirf ye 4 yaad rakho (interview ke liye)
1 = Element  |  3 = Text  |  8 = Comment  |  9 = Document
Trick: "Every Tree Contains Documents" → 1, 3, 8, 9 (rough pattern but helps recall!)
Live example — nodeTypes visible karo
document nodeType: 9
├─<!DOCTYPE html> nodeType: 10
└─<html> nodeType: 1 (Element)
└─<body> nodeType: 1 (Element)
├─<h1> nodeType: 1
└─"Hello World" nodeType: 3 (Text)
├─<!-- navbar here --> nodeType: 8 (Comment)
└─"\n " (whitespace) nodeType: 3 (Text!)
nodeType in action 
// ── nodeType check karna ────────────────────────────
const el = document.querySelector('h1');
console.log(el.nodeType);    // 1  → Element
console.log(el.nodeName);    // "H1" (uppercase tag name)
console.log(el.nodeValue);   // null (elements ka nodeValue null hota hai)

// ── Text node ────────────────────────────────────────
const textNode = el.firstChild;  // whitespace bhi text node ho sakta hai!
console.log(textNode.nodeType);  // 3
console.log(textNode.nodeValue); // "Hello World" (actual text content)

// ── Comment node ─────────────────────────────────────
const comment = document.createComment('My comment');
console.log(comment.nodeType);  // 8
console.log(comment.nodeValue); // "My comment"

// ── Document node ────────────────────────────────────
console.log(document.nodeType); // 9
console.log(document.nodeName); // "#document"

// ── Real use: childNodes iterate karte waqt filter karna
const body = document.body;
body.childNodes.forEach(node => {
  if (node.nodeType === 1) {          // sirf Elements
    console.log('Element:', node.tagName);
  } else if (node.nodeType === 3) {   // Text nodes (whitespace!)
    console.log('Text:', node.nodeValue.trim());
  }
});
🏷️

nodeName

Element → uppercase tag name ("DIV")
Text → "#text"
Comment → "#comment"
Document → "#document"

💾

nodeValue

Element → null (elements ki value nahi hoti)
Text → actual text content
Comment → comment text
Document → null

🔢

nodeType

Integer constant. Use the Node.ELEMENT_NODE constants for readable code:
Node.TEXT_NODE === 3 ✅

⚠️

Whitespace Trap!

Newlines aur spaces between tags bhi Text Nodes create karte hain (nodeType 3). Isliye childNodes surprising results de sakta hai!

⚠️ Common Interview Trap — childNodes vs children:
childNodes returns ALL node types including Text (3) and Comment (8) nodes.
children returns ONLY Element nodes (nodeType 1).
Agar tumhare paas <ul>\n <li>...</li>\n</ul> hai toh childNodes.length will be 3 (text + li + text), but children.length will be 1!
Bonus Q How many types of nodes does the DOM have? Name the important ones. +
DOM has 12 node types total (defined in the Node interface), but in practice only a few matter:

— nodeType 1 (Element Node): Every HTML tag — <div>, <p> etc.
— nodeType 3 (Text Node): Text content inside elements, including whitespace and newlines.
— nodeType 8 (Comment Node): HTML comments <!-- -->
— nodeType 9 (Document Node): The document object itself, root of the tree.
— nodeType 11 (DocumentFragment): In-memory container used for performance optimization.

You can check with node.nodeType or use readable constants like Node.ELEMENT_NODE.

Selecting Elements

🎯 Real-life analogy: Selectors are like calling someone by their name, nickname, or job role. getElementById = calling by exact name. querySelector = calling by any identifier (CSS magic!).
Method Returns Use When Speed
getElementById('id') Single Element / null You know the exact id ⚡ Fastest
getElementsByClassName('cls') HTMLCollection (live) Multiple elements by class 🔥 Fast
getElementsByTagName('div') HTMLCollection (live) All elements of a tag type 🔥 Fast
querySelector('css') First match / null Complex CSS selectors 🚶 Moderate
querySelectorAll('css') NodeList (static) All matches with CSS selector 🚶 Moderate
JavaScript 
// ── getElementById ──────────────────────────────────
const title = document.getElementById('main-title');
// Returns: element or null (agar id exist nahi karta)

// ── querySelector (Most Flexible!) ───────────────────
document.querySelector('.btn');        // class
document.querySelector('#hero');       // id
document.querySelector('div > p');    // nested
document.querySelector('input[type="text"]'); // attribute

// ── querySelectorAll (Returns NodeList) ──────────────
const cards = document.querySelectorAll('.card');
cards.forEach(card => console.log(card)); // forEach works directly!

// ── INTERVIEW TRAP: HTMLCollection vs NodeList ───────
const live = document.getElementsByClassName('item'); // LIVE
const snap = document.querySelectorAll('.item');    // STATIC snapshot
⚠️ Common Interview Trap: HTMLCollection is live — DOM update ho toh automatically update ho jaata hai. NodeList from querySelectorAll is static — ek snapshot hai. And NodeList has forEach() but not map()/filter() directly!

NodeList & HTMLCollection

📦 Analogy: Socho tumne ek class ke sab students ki list banayi. HTMLCollection ek live attendance register hai — naya student join kare toh automatically update ho jaata hai. NodeList ek printed snapshot hai — print ke waqt jo tha wahi rahega, baad mein kuch bhi badlo, list nahi badlegi. Dono list hain, par behavior alag hai!
HTMLCollection
🔴 Live — auto-updates
📌 Returns from getElementsBy*
🏷️ Only Element nodes (nodeType 1)
🔢 Access by index or name: col[0]
❌ No forEach natively
❌ No map / filter / reduce
✅ Has .length
✅ Array mein convert kar sakte ho
NodeList
📸 Static — frozen snapshot (usually)
📌 Returns from querySelectorAll
🏷️ All node types (elements, text, comments)
🔢 Access by index only: list[0]
✅ Has forEach natively
❌ No map / filter / reduce directly
✅ Has .length
✅ Array mein convert kar sakte ho
⚡ Live vs Static — Visually Samjho
HTMLCollection (Live)
const items = document
  .getElementsByClassName('item');
// Abhi 3 .item hain
console.log(items.length); // 3

// Naya element add karo
document.body.appendChild(newItem);
console.log(items.length); // 4 ✅ auto!
NodeList (Static)
const items = document
  .querySelectorAll('.item');
// Abhi 3 .item hain
console.log(items.length); // 3

// Naya element add karo
document.body.appendChild(newItem);
console.log(items.length); // 3 ❌ stale!
Array mein convert karna — Must Know! 
// ── Problem: dono par map/filter direct kaam nahi karta
const col = document.getElementsByClassName('card'); // HTMLCollection
const nl  = document.querySelectorAll('.card');      // NodeList

col.map(el => el);    // ❌ TypeError: col.map is not a function
nl.map(el => el);     // ❌ TypeError: nl.map is not a function

// ── Solution 1: Array.from() — Most Readable ─────────
const arr1 = Array.from(col);
const arr2 = Array.from(nl);
arr1.map(el => el.textContent); // ✅ works now!

// ── Solution 2: Spread operator — Short & Clean ───────
const arr3 = [...col];
const arr4 = [...nl];

// ── NodeList mein forEach directly milta hai ──────────
nl.forEach(el => console.log(el));  // ✅ NodeList ka forEach kaam karta hai
// col.forEach(...)  ❌ HTMLCollection mein nahi hota

// ── Access by index (dono mein) ───────────────────────
col[0];  // ✅
nl[0];   // ✅
col.item(0);  // ✅ .item() method dono mein hai

// ── HTMLCollection: name se bhi access kar sakte ho ──
// <input name="username"> ho toh:
document.forms[0].namedItem('username'); // HTMLCollection feature
Feature HTMLCollection NodeList
Live / Static 🔴 Live (auto-updates) 📸 Static (usually)
Returns from getElementsByClassName, getElementsByTagName, children querySelectorAll, childNodes
Node types included Element nodes only (type 1) All node types (1, 3, 8...)
forEach() ❌ Not available ✅ Available
map / filter / reduce ❌ Not available ❌ Not available
Access by index ✅ col[0] ✅ nl[0]
Access by name ✅ namedItem('x') ❌ No
Convert to Array Array.from(col) or [...col] Array.from(nl) or [...nl]
Is Array? ❌ Array-like object ❌ Array-like object
💡 Exception — Live NodeList bhi hoti hai!
Trick question alert 🎤: Zyaadatar NodeList static hoti hai, lekin childNodes se jo NodeList milti hai wo LIVE hoti hai! Interviewer ye pooch sakta hai.
Live NodeList: element.childNodes
Static NodeList: document.querySelectorAll()
Bonus Q HTMLCollection aur NodeList mein kya difference hai? Dono array kyun nahi hain? +
HTMLCollection: Live, auto-updates jab DOM change ho. Sirf Element nodes (nodeType 1). getElementsByClassName etc. se milti hai. forEach nahi.

NodeList: Mostly static snapshot. Saare node types. querySelectorAll se milti hai. forEach available hai. Exception: childNodes live NodeList deta hai.

Array kyun nahi hain? Ye DOM Web API ke objects hain, JS Array nahi. Unhe "array-like" kehte hain — indexed access aur .length hai, lekin Array prototype ke methods nahi. Array.from() ya spread [...] se real array banaao, phir saare methods milenge.

DOM Manipulation

✏️ Analogy: DOM manipulation = living room ke furniture ko rearrange karna, naya furniture laana, ya kuch hata dena — sab kuch ghar (page) ke andar hi, bina nayi jagah shift kiye!
Content & Text 
const el = document.querySelector('.box');

// ── Text content ─────────────────────────────────────
el.textContent = 'Hello World';    // Safe: treats as plain text
el.innerHTML   = '<b>Bold!</b>';  // Parses HTML (⚠️ XSS risk!)
el.innerText   = 'Visible only';  // Respects CSS visibility

// ── Attributes ───────────────────────────────────────
el.getAttribute('href');           // get
el.setAttribute('href', '/');     // set
el.removeAttribute('disabled');   // remove
el.hasAttribute('class');        // check → boolean

// ── Classes (Modern Way — use classList!) ────────────
el.classList.add('active');
el.classList.remove('active');
el.classList.toggle('dark');       // add ↔ remove
el.classList.contains('btn');    // → boolean
el.classList.replace('old', 'new');

// ── Styles ───────────────────────────────────────────
el.style.color = 'red';
el.style.backgroundColor = 'blue'; // camelCase!!
el.style.setProperty('--my-var', '10px'); // CSS custom vars
Creating & Inserting Elements 
// ── Create element ───────────────────────────────────
const newDiv = document.createElement('div');
newDiv.textContent = 'I am new!';
newDiv.className = 'card';

// ── Append / Prepend (Modern — preferred!) ───────────
const parent = document.querySelector('.container');
parent.append(newDiv);      // add at END (accepts strings too!)
parent.prepend(newDiv);     // add at BEGINNING
parent.before(newDiv);      // sibling BEFORE parent
parent.after(newDiv);       // sibling AFTER parent

// ── Old Way (still in interviews!) ───────────────────
parent.appendChild(newDiv);            // OLD — only nodes
parent.insertBefore(newDiv, sibling);  // OLD

// ── Remove element ───────────────────────────────────
newDiv.remove();                  // Modern — direct!
parent.removeChild(newDiv);       // Old way

// ── Clone element ────────────────────────────────────
const clone = newDiv.cloneNode(true); // true = deep clone (children bhi)
💡 Memory Trick — textContent vs innerHTML vs innerText
textContent = exactly what's in the source, HTML tags as literal text, fast.
innerHTML = parses HTML, powerful but XSS danger with user input.
innerText = what user sees (respects CSS display:none), slow (triggers reflow).
Yaad karo: For user data → always textContent. Never trust innerHTML with user input!

Events & Event Handling

⚡ Analogy: Events are like doorbells 🔔. Aapne ek doorbell (listener) lagayi hai — jab bhi koi press kare (event happen ho), aapka code run ho jaata hai. EventListener = doorbell wire. Callback = aapka response (door kholna).
addEventListener — Sahi Tarika 
const btn = document.querySelector('#myBtn');

// ── 3 Ways (only ONE is correct in 2024!) ────────────
// ❌ Old Way 1 — HTML attribute (don't use)
// <button onclick="doSomething()">

// ❌ Old Way 2 — only one handler per event
btn.onclick = () => console.log('clicked');

// ✅ Modern Way — multiple handlers, full control
btn.addEventListener('click', function(event) {
  console.log(event);          // full event object
  console.log(event.target);  // element that was clicked
  console.log(event.type);    // 'click'
  event.preventDefault();     // default action rokta hai (form submit, link etc)
  event.stopPropagation();    // bubbling rokta hai
});

// ── Remove listener ──────────────────────────────────
const handler = (e) => console.log(e);
btn.addEventListener('click', handler);
btn.removeEventListener('click', handler); // same reference chahiye!

Event Propagation — Bohot Important!

⬇️
1. Capture Phase (Trickling Down)
Event starts from window → goes DOWN to target element
Use addEventListener('click', fn, true) to listen in capture phase. Rarely needed in real code.
›
🎯
2. Target Phase
Event reaches the actual element that was interacted with
event.target is this element. Both capture & bubble listeners fire here.
›
⬆️
3. Bubble Phase (Bubbling Up)
Event goes UP from target → back to window
This is the DEFAULT. Most events bubble. focus, blur, scroll do NOT bubble. Use event.stopPropagation() to stop bubbling.
›
Event Delegation — Performance Trick! 🔥 
// ❌ BAD — 1000 buttons = 1000 listeners = memory hog
document.querySelectorAll('.btn').forEach(btn => {
  btn.addEventListener('click', handleClick);
});

// ✅ GOOD — Event Delegation (bubbling ka fayda uthao!)
// Ek parent listener, all children handle karo
document.querySelector('.btn-container').addEventListener('click', function(e) {
  if (e.target.classList.contains('btn')) {
    console.log('Button clicked:', e.target.textContent);
  }
});
// Dynamic elements par bhi kaam karta hai! ✨
🎤 Interview Gold — Event Delegation
"Why use event delegation?"
1. Performance — ek listener, 1000 nahi
2. Dynamic elements — baad mein add hone wale elements bhi automatically handle ho jaate hain
3. Less memory — garbage collection friendly
Example: Todo list mein har li par listener lagate ho ya ul par ek? → ul par ek!

DOM Traversal

🗺️ Analogy: Traversal = family tree navigate karna. Parent kya hai? Siblings kaun hain? Children kaunse? Exact CSS selector na pata ho tab traversal se dhoondo!
Property Returns Includes Text Nodes?
el.parentNode Parent (any node type) Yes
el.parentElement Parent element only No (preferred!)
el.children HTMLCollection of child elements No ✅
el.childNodes NodeList (all nodes) Yes (text, comments)
el.firstElementChild First child element No ✅
el.lastElementChild Last child element No ✅
el.nextElementSibling Next sibling element No ✅
el.previousElementSibling Previous sibling element No ✅
el.closest('.class') Nearest ancestor matching CSS No ✅
💡 Rule of Thumb
Jab bhi choice ho — use the Element versions (parentElement, children, nextElementSibling) instead of Node versions. They skip text nodes and whitespace, giving you only actual HTML elements. Less surprises!

Interview Q&A 🎤

Q1 What is the DOM? Is it part of JavaScript? +
DOM is NOT part of JavaScript. It is a Web API provided by the browser. JavaScript uses the DOM API to interact with HTML. The DOM represents the page as a tree of nodes (objects). Even if you run JS in Node.js, you don't have a DOM — that's browser-specific. DOM is defined by W3C/WHATWG standards, not ECMAScript.
Q2 Difference between getElementById and querySelector? +
getElementById: only works with IDs, fastest (direct hash lookup), returns null if not found.

querySelector: accepts any CSS selector, slightly slower (has to parse selector), more flexible. Returns first match or null.

Prefer getElementById when you have an ID — it's the fastest. Use querySelector when you need complex selectors or class-based selection.
Q3 What is event bubbling and how does stopPropagation work? +
Event Bubbling: When an event fires on an element, it travels UP the DOM tree — first the target, then its parent, then grandparent, all the way to window.

stopPropagation(): Stops the event from bubbling up further. Parent elements won't receive the event.

stopImmediatePropagation(): Stops bubbling AND prevents other listeners on the SAME element from firing.

Note: focus, blur, scroll events do NOT bubble.
Q4 What is event delegation? When and why use it? +
Event delegation = adding a single event listener on a parent to handle events from all its children, using event bubbling.

Why:
— Better performance (one listener vs hundreds)
— Works for dynamically added elements automatically
— Less memory usage

How: Use event.target inside the parent's listener to identify which child was actually clicked.
Q5 Difference between innerHTML, textContent, and innerText? +
textContent: Returns/sets the raw text content. HTML tags treated as literal text. Fast, doesn't trigger reflow, XSS safe.

innerHTML: Parses HTML. Use for inserting actual HTML. XSS dangerous with user input — never do el.innerHTML = userInput.

innerText: Aware of CSS styling. Hides text in display:none elements. Triggers reflow (slow). Not available in Node.js environments.
Q6 What is the difference between e.target and e.currentTarget? +
e.target: The element that actually triggered the event (where you clicked/typed).

e.currentTarget: The element where the event listener is attached.

In event delegation, these are different! If you click a <li> inside a <ul> listener: e.target = li, e.currentTarget = ul.
Q7 What does preventDefault() do? Give an example. +
preventDefault() stops the browser's default behavior for that event.

Examples:
— Form submit: Stops page reload
— Link click: Stops navigation to href
— Right click: Stops context menu
— Keydown: Stops character from typing

Important: It does NOT stop bubbling — that's stopPropagation()'s job.
Q8 What is document fragment and why use it? +
A DocumentFragment is a lightweight container that exists in memory — NOT in the actual DOM.

Why use it: When you're appending many elements, each appendChild triggers a reflow (expensive!). With DocumentFragment, you build your tree in memory, then append once — only ONE reflow.

const frag = document.createDocumentFragment();
items.forEach(item => frag.appendChild(createItem(item)));
list.appendChild(frag); // ONE reflow ✅
