This section tracks hackathon showcases and references separately from normal projects.

This folder should stay lightweight and tracking-focused.

Hackathon code should usually live in a dedicated GitHub repository because it is competition-style work with its own starter code, schema, deployment, README, and submission links.

## What Goes Here

- hackathon name and date
- separate GitHub repo link
- live deployment link
- demo video link
- submission link
- short notes if needed

---

## What Does Not Go Here

- full hackathon starter code
- `node_modules`
- generated build files
- private `.env` files
- temporary experiment files

Actual hackathon source code should stay in its own dedicated repo, while this repo keeps only the summary and redirect links.
